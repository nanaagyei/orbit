ORBIT — FULL UI / UX DESIGN SYSTEM

Design goal:
Make Orbit feel like a calm, serious operating system for ML practitioners—
not a dashboard, not a CRM, not a flashy AI toy.

1️⃣ Product Feel & Design Philosophy
Orbit should feel like:

A research notebook

A thoughtful relationship ledger

A quiet command center

A tool built by engineers, for engineers

Orbit should NOT feel like:

A social network

A sales CRM

A Notion clone

A flashy AI demo

2️⃣ Core UX Principles (Non-Negotiable)
1. Calm > Clever

No animations unless they add clarity

No dopamine-driven UI

No gamification noise

2. Next Action First

Every screen should answer:

“What is the most meaningful thing I can do next?”

3. Progressive Disclosure

Beginner-friendly by default

Power-user depth revealed when needed

Never overwhelm

4. Keyboard-First

Orbit should feel fast:

⌘K for actions

⌘/ for search

⌘Enter to save

⌘1–⌘6 to navigate

5. Local-First Confidence

Subtle signals:

“Saved locally”

“Offline ready”

“Export anytime”

3️⃣ Visual Design System
Color Palette (Minimal, Professional)

Background:

#0F1115 (dark) or #FAFAFA (light)

Primary Accent:

Muted blue or slate (#4F6EF7 or similar)

Secondary:

Neutral gray scale

Semantic Colors:

Green (completed)

Amber (due soon)

Red (overdue)

No gradients. No neon. No AI purple.

Typography

Primary font: Inter / Geist / SF Pro

Hierarchy:

Page titles: medium, restrained

Body text: readable, slightly dense

Metadata: lighter gray

Orbit should feel like Linear + GitHub + a research notebook had a child.

Layout

Max width: ~1200px

Whitespace: Generous but intentional

Cards: Soft borders, not heavy shadows

Tables: Dense, scannable, sortable
